================================================================================
                      ATTACK SURFACE ASSESSMENT
================================================================================

Attack Vector 1: BATCH QUERY ENUMERATION
  Method: Submit multiple operations in single GraphQL request
  Impact: Rapidly extract data across multiple entities
  Mitigation: Rate limiting per request, query complexity scoring

Attack Vector 2: TYPE UNION EXPLOITATION
  Method: Use "... on Member" syntax to probe user types
  Impact: Identify user roles without permission verification
  Mitigation: Field-level authorization, role-based query filtering

Attack Vector 3: NESTED QUERY TRAVERSAL
  Method: Deeply nested queries to extract related data
  Example: viewer → leagues → teams → matches → stats
  Impact: Full organization hierarchy extraction
  Mitigation: Query depth limits, field selection whitelisting

Attack Vector 4: SCHEMA INTROSPECTION
  Method: { __schema { types { name fields { name } } } }
  Impact: Complete API reverse-engineering
  Mitigation: Disable introspection in production

Attack Vector 5: CROSS-USER DATA ACCESS
  Method: Modify JWT sub claim or iterate user IDs
  Impact: Access other users' leagues, teams, stats
  Mitigation: Validate token signature, verify sub claim

Attack Vector 6: MUTATION EXPLOITATION
  Method: Unknown mutations may allow data modification
  Impact: Create/update/delete users, leagues, matches
  Mitigation: Require strong authorization for mutations

Attack Vector 7: QUERY COMPLEXITY ABUSE
  Method: Very deeply nested or complex queries
  Impact: Denial of service via resource exhaustion
  Mitigation: Query complexity analysis and limits
